## üß† How `gen_api_pages.py` Fits In

- `gen_api_pages.py` is responsible for **materializing endpoint-level docs** under `docs/api/`, typically by:
  - Reading an OpenAPI schema (e.g., `docs/api/openapi.json` or the live FastAPI app)
  - Generating one Markdown page per endpoint or tag
- This allows you to:
  - Keep **API reference** docs in sync with the code
  - Link from `API.md` to the detailed pages under `docs/api/`

You can later extend the same pattern to also:

- Regenerate this `AUTO_INDEX.md`
